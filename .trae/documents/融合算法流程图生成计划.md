# 融合算法流程图生成计划

您提供的 Mermaid 代码描述了 **双模型加权融合** 的决策流程。为了保持报告风格的一致性，我将继续使用 `matplotlib` 绘制一张 **学术级算法流程图**，而非简单的 Mermaid 框图。

## 绘图设计方案
这张图将展示数据流如何在两个模型之间并行处理，最终通过动态权重机制融合。

### 1. 视觉布局 (垂直流程)
*   **顶部输入**：监测数据（Input）。
*   **并行分支**：
    *   左路：BiLSTM 模型 -> 预测值 A。
    *   右路：Stacking 模型 -> 预测值 B。
*   **核心逻辑**：趋势一致性校验（菱形判定框）。
*   **权重调节**：动态权重分配模块。
*   **底部输出**：最终加权结果。

### 2. 配色方案 (算法绿风格)
为了区别于架构图的蓝色系，算法图将采用 **科技绿/深灰** 配色，强调逻辑与计算。
*   **输入/输出**：#4CAF50 (标准绿)
*   **模型/处理**：#FFFFFF (白底黑边)
*   **判定/逻辑**：#FF9800 (橙色，强调决策点)
*   **连线**：#455A64 (深灰)

### 3. 技术实现 (generate_assets.py)
新增 `generate_fusion_algorithm_flow()` 函数：
*   使用 `patches.FancyBboxPatch` 绘制圆角矩形（处理步骤）。
*   使用 `patches.Polygon` 绘制菱形（判定条件）。
*   使用 `patches.Circle` 绘制融合节点。
*   绘制平滑的贝塞尔曲线连接各节点。
